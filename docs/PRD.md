# 핏저니 (FitJourney) PRD
## 제품 요구사항 정의서

---

## 1. 제품 개요

- **앱 이름**: 핏저니 (FitJourney)
- **슬로건**: "투약과 함께하는 건강한 변화"
- **비전**: GLP-1 투약자가 투약·체중·운동·식사를 한곳에서 기록하고 시각적으로 추적하여 건강한 변화를 직접 체감하는 개인 건강 관리 도구
- **핵심 가치**:
  - 서버 없이 브라우저만으로 동작 (프라이버시 보장)
  - GLP-1 투약 주기 자동 추적
  - 투약-체중-운동-식사 4축 데이터 시각화
  - 목표 설정과 마일스톤으로 동기 부여
- **범위 밖(Out of Scope)**: 의료 자문, 처방 관리, 소셜 기능, 서버 동기화, PWA

---

## 2. 타겟 사용자

- **1차**: GLP-1 계열 약품(마운자로/위고비/삭센다/오젬픽) 투약 중인 한국 사용자
- **2차**: 투약 예정자, 투약 완료 후 유지 관리 중인 사용자

| 페르소나 | 설명 | 핵심 니즈 |
|---------|------|----------|
| 김민지 (30대, 직장인) | 마운자로 5mg 투약 3개월차 | 빠른 입력, 모바일 최적화, 진행률 시각화 |
| 이준호 (40대, 운동 병행) | 위고비 + 헬스 병행 | 인바디 데이터, 운동 상세 기록, 통계 |
| 박서연 (50대, 건강 관리) | 삭센다 투약, 식사 조절 중심 | 칼로리 목표, 부작용 추적, 데이터 내보내기 |

## 4. 기능 요구사항

### 4-A. 기존 기능 개선

| ID | 기능 | 개선 사항 |
|----|------|----------|
| F-01 | 투약 기록 | **편집 추가**, 약품명 필드(마운자로/위고비/삭센다/오젬픽/기타), 투약 주기 커스텀 |
| F-02 | 체중/인바디 기록 | **편집 추가** |
| F-03 | 운동 기록 | **편집 추가**, 소모 칼로리 필드 |
| F-04 | 식사 기록 | **편집 추가**, 영양소(탄/단/지) 선택 입력 |
| F-05 | 대시보드 | 목표 진행률 위젯, 주간 요약 추가 |
| F-06 | 체중 차트 | 기간 필터(1주/1개월/3개월/전체) |

### 4-B. 신규 기능

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| F-10 | 기록 편집 | 모든 기록에 bottom-sheet 모달 기반 편집 UI | P0 |
| F-11 | 약품 선택 | 투약 기록에 약품명 필드, 약품별 용량 자동 변경 | P0 |
| F-12 | 목표 설정 & 진행률 | 체중/주간 운동/일일 칼로리 목표 + 프로그레스 바 | P0 |

| F-15 | 데이터 내보내기/가져오기 | JSON 백업 + CSV 변환 + 차트 이미지 캡처 | P1 |
| F-16 | 다크/라이트 테마 전환 | CSS 변수 세트 전환, 시스템 설정 자동 감지 | P2 |
| F-17 | 주간/월간 리포트 | 기간별 체중 변화, 운동, 칼로리 요약 카드 | P2 |
| F-18 | 데스크톱 레이아웃 | 768px+ 2컬럼, 1024px+ 사이드바 네비 | P2 |

### 4-C. 편집 모달 UX

- 각 record-item에 "편집" 버튼 추가 (삭제 옆)
- 클릭 → bottom-sheet 모달 (`.modal-overlay` + `.modal-sheet`)
- `Storage.getById()` → 폼 채움 → `Storage.update()` → 모달 닫기 → `renderList()`
- 모달 외부 클릭 시 닫기

### 4-D. 설정 페이지

- **진입**: 헤더 우측 ⚙️ 아이콘 (탭 5개 유지)
- **내용**: 목표 설정, 테마 전환, 데이터 내보내기/가져오기, 마일스톤 관리, 약품/투약 주기 설정

## 5. 데이터 모델

### 기존 키 유지 (변경 금지)

| 키 | 설명 |
|----|------|
| `mj_mounjaro` | 투약 기록 |
| `mj_body` | 체중/인바디 기록 |
| `mj_exercise` | 운동 기록 |
| `mj_diet` | 식사 기록 |

### 신규 키

| 키 | 설명 |
|----|------|
| `mj_goals` | 목표 설정 `{ weightGoal, exerciseGoal, calorieGoal }` |
| `mj_milestones` | 마일스톤 배열 `{ id, type, title, achievedAt, targetDate? }` |
| `mj_settings` | 설정 `{ theme, drugName, doseInterval }` |
| `mj_version` | 스키마 버전 (마이그레이션 판단용) |

### 기존 레코드 필드 확장

- **투약**: `drugName` 추가 (기존 데이터는 `'mounjaro'` 기본값)
- **운동**: `caloriesBurned` 추가
- **식사**: `carbs`, `protein`, `fat` 추가
- **마이그레이션**: `storage.js`에 `migrate()` 함수, `mj_version`으로 중복 실행 방지

## 6. UI/UX 요구사항

- 모바일 우선, 한 손 조작 최적화
- 최소 탭으로 빠른 기록 (기본값 자동 채움)
- 시각적 피드백 (토스트, 프로그레스 바)
- 라이트 테마: `:root`에 라이트, `[data-theme="dark"]`에 다크 변수
- 데스크톱: 768px+ 2컬럼, 1024px+ 사이드바 네비
